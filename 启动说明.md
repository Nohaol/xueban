# 本机启动说明

这份说明只针对当前电脑和当前项目目录：

```powershell
C:\Users\20245\WeChatProjects\miniprogram-1
1. 进入项目目录
cd C:\Users\20245\WeChatProjects\miniprogram-1
2. 确认使用项目虚拟环境
必须使用 .venv-backend，不要用系统 Python 或 Anaconda Python。
.\.venv-backend\Scripts\python.exe -c "import mediapipe; print('mediapipe ok')"
输出 mediapipe ok 就说明环境正确。
3. 启动后端和前端
.\.venv-backend\Scripts\python.exe -m uvicorn backend.main:app --host 127.0.0.1 --port 8000
这个 PowerShell 窗口不要关闭。
4. 打开前端
浏览器访问：
http://127.0.0.1:8000/v2
前端由后端托管，不需要另外启动 npm。
5. 确认正在使用 MediaPipe
另开一个 PowerShell，在项目根目录运行：
.\.venv-backend\Scripts\python.exe -c "import urllib.request,json; print(json.load(urllib.request.urlopen('http://127.0.0.1:8000/state'))['engineMode'])"
期望输出：
mediapipe
如果是 camera，说明走了后备识别框路径。
如果是 mock，说明后端没有读到摄像头画面。
6. 如果 8000 端口被占用
netstat -ano | Select-String ':8000'
找到 LISTENING 那行最后的 PID，例如 49652，然后：
Stop-Process -Id 49652
再重新运行启动命令。
7. 关闭后端
前台运行时按：
Ctrl + C
后台进程则先查 PID：
netstat -ano | Select-String ':8000'
再停止：
Stop-Process -Id <PID>
8. 常见现象
看到绿色人脸框
说明不是 MediaPipe，而是后备 Haar 识别框路径。优先检查是不是用错 Python 环境。
正确命令必须是：
.\.venv-backend\Scripts\python.exe -m uvicorn backend.main:app --host 127.0.0.1 --port 8000
不要用：
python -m uvicorn ...
也不要用 Anaconda Python。
页面是 mock 数据
说明后端没有读到摄像头画面。在摄像头能正常打开的前提下，重启后端即可。
```
