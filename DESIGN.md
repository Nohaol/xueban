# 小智伴学家长端网页重设计方案

## 设计判断

Reading this as: 家长端学习陪伴控制台 redesign for 家长用户, with a trust-first and calm product UI language, leaning toward restrained app-shell plus accessible dashboard patterns.

这个网页的核心任务不是展示所有指标，而是帮助家长判断：

> 现在要不要介入孩子的学习状态？

因此新版界面应从“监控大屏”转向“陪伴决策工作台”。实时视频、专注分、AI 建议、提醒控制、趋势图和设备配置都重要，但不应该在同一屏抢主次。

## 设计参数

- `DESIGN_VARIANCE: 4`
- `MOTION_INTENSITY: 3`
- `VISUAL_DENSITY: 5`

理由：

- 涉及儿童学习、摄像头和家长干预，界面需要可信、安静、克制。
- 不适合炫技式大屏、AI 紫色渐变、玻璃拟态和高频动效。
- 信息密度可以中等，但首页必须围绕“是否提醒”收敛。

## 产品定位

新版产品应表达：

- 不是监视孩子，而是辅助家长温和陪伴。
- 不是展示算法能力，而是给出可执行判断。
- 不是工程调试台，而是家长可以放心使用的日常工具。

首页最重要的表达不应是：

> 综合专注度 82

而应是：

> 当前无需打扰，继续观察

或：

> 适合轻提醒一次

## 信息架构

一级导航建议保留 4 个入口：

1. 看护
2. 建议
3. 复盘
4. 设备

不要把所有功能都做成一级入口。家长不是后台运营用户，入口越多越像工程系统。

### 看护

默认首页，只回答一个问题：现在要不要介入？

保留内容：

- 实时视频画面
- 当前状态
- 一句话决策
- 主操作按钮
- 最近 3 条事件
- 今日简短摘要

移出内容：

- 视频源表单
- AI 详细解释
- 长时间线
- 趋势图细节
- 系统配置项

建议布局：

```text
顶部栏：孩子 / 当前学习会话 / 摄像头在线状态

主区域左侧：
实时视频

主区域右侧：
当前状态
一句话判断
主按钮：发送提醒 / 继续观察
次按钮：刷新视频 / 智能分析

底部：
最近 3 条事件
今日简短摘要
```

### 建议

承载 AI 分析、小智话术和提醒记录。

核心模块：

- 当前建议结论
- 判断依据
- 小智可朗读话术
- 发送前确认
- 最近提醒记录
- 话术模板库

话术模板建议分组：

- 鼓励型
- 坐姿型
- 回座型
- 休息型

AI 建议不应做成聊天气泡。它更像一张“决策单”，需要短、清楚、有理由。

### 复盘

承载历史趋势和学习会话，不做炫技图表。

核心模块：

- 今日专注曲线
- 离座次数
- 分心波动次数
- 学习会话列表
- 本次学习摘要

暂不建议优先实现：

- 复杂雷达图
- 大面积热力图
- 多层级钻取分析
- 过密状态环图

复盘页的目标是让家长知道“今天整体怎么样”，不是让家长研究算法。

### 设备

承载工程性和配置性内容。

核心模块：

- 视频源列表
- 添加 IP Webcam / RTSP / HTTP snapshot
- 视频源连通性测试
- 小智 MCP 状态
- DeepSeek 配置
- 分析阈值设置
- mock 模式

这页允许比首页更密，因为用户进入这里就是为了解决配置问题。

## 视觉系统

### 气质

关键词：

- 安静
- 可信
- 温和
- 有边界感

避免：

- 科技大屏感
- 儿童卡通感
- AI 紫色渐变
- 大量玻璃拟态
- 过度圆角和漂浮阴影

### 色彩

建议使用单主色系统。

```css
:root {
  --color-bg: #f6f7f4;
  --color-surface: #ffffff;
  --color-surface-muted: #eef2ed;
  --color-text: #1f2a24;
  --color-text-muted: #69746d;
  --color-border: #dde3dc;

  --color-accent: #3e7c59;
  --color-focus: #63b383;
  --color-warning: #d99a3d;
  --color-danger: #c75c5c;
  --color-info: #4d84a8;
}
```

规则：

- 一个主 accent，全站一致。
- 状态色只用于状态表达，不做装饰。
- 首页不要多色卡片并排抢注意力。
- 深色模式可以后置，不要第一版同时做两套导致质量分散。

### 字体

优先使用系统中文字体。

```css
font-family:
  "PingFang SC",
  "Microsoft YaHei",
  system-ui,
  -apple-system,
  BlinkMacSystemFont,
  sans-serif;
```

数字和计时器建议使用 tabular numbers：

```css
font-variant-numeric: tabular-nums;
```

### 形状

统一半径系统：

- 页面容器：不做大圆角
- 卡片：`8px`
- 按钮：`8px`
- 输入框：`8px`
- 状态 pill：`999px`

不要混用大圆角卡片、胶囊按钮和直角输入框。

### 卡片

卡片只用于真正需要分组的信息。

首页卡片数量控制在 4 个以内。能用留白、分隔线、标题层级解决的，不要强行做卡片。

## 交互原则

### 主按钮

每个页面只保留一个主操作。

看护页主操作示例：

- 当前正常：继续观察
- 轻度分心：发送轻提醒
- 离座：稍后提醒
- 超时离座：确认休息状态

### 提醒发送

提醒不应一键冲出去，建议采用确认流：

1. AI 或本地规则生成话术
2. 家长查看话术
3. 家长点击发送
4. 记录发送结果
5. 1 到 2 分钟后观察状态是否改善

### 状态反馈

需要覆盖：

- 加载中
- 连接失败
- 摄像头离线
- AI 调用失败
- 小智未连接
- 空数据
- 已发送成功

不要只设计成功状态。

### 动效

动效强度保持低。

允许：

- 页面切换轻微淡入
- 按钮按压反馈
- 状态变化 150 到 250ms 过渡
- 图表数据更新的轻微过渡

避免：

- 背景持续动画
- 大面积视差
- 自动轮播
- 装饰性闪光

## 功能扩充优先级

### 第一优先级

- 学习会话开始和结束
- AI 话术确认后发送
- 提醒冷却时间
- 摄像头源测试
- 今日专注摘要

### 第二优先级

- 话术模板库
- 自动提醒策略
- 多孩子档案
- 周报和月报
- 隐私模式

### 暂不优先

- 复杂图表系统
- 多维雷达图
- 大屏模式
- 过度 AI 解释
- 自动化强干预

## 技术落地建议

第一步不要直接替换现有页面。建议新增 v2 文件：

- `backend/static/parent-console-v2.html`
- `backend/static/parent-console-v2.css`
- `backend/static/parent-console-v2.js`

先完成静态和前端交互原型，再接入现有接口：

- `GET /state`
- `GET /sources`
- `POST /sources/select`
- `POST /sources/network`
- `POST /control`
- `GET /ai/advice`
- `WS /ws`
- `GET /video_feed`

这样可以保留当前可演示版本，同时快速验证新版信息架构。

## 交付检查

上线或替换前需要检查：

- 首页是否只回答“现在要不要介入”
- 是否每页只有一个主操作
- 首页是否移除了配置表单和复杂图表
- 所有按钮文字是否清晰且不换行
- 状态色是否只用于状态表达
- 视频离线、AI 失败、小智未连接是否都有明确状态
- 移动端是否没有横向滚动
- 字号和触控区域是否适合家长快速操作
- 是否避免了 AI 紫色渐变、玻璃拟态、过度动效和堆卡片
