# 小智伴学：USB 断开与局域网使用说明

## 结论

日常使用时可以断开机器人与电脑之间的 USB 连接。USB 目前只承担首次烧录、串口调试和临时供电；主动播报、三学段切换、中文对话和摄像头分心提醒，均通过局域网完成。

断开 USB 后需要同时满足：

1. 机器人使用独立电源供电。
2. 电脑与机器人接入同一个可互相访问的 Wi-Fi、路由器或手机热点。
3. 电脑上的五个本地服务端口正常：`8000`、`1883`、`18000`、`18003`、`18007`。
4. 固件中的 OTA 服务地址指向电脑当前局域网 IP。

## 为什么电脑 IP 会变化

当前电脑使用 DHCP 自动获取地址。路由器、校园网或手机热点会在重新连接、租约续期、切换网络、设备重启后重新分配地址。因此电脑可能从 `10.143.97.5` 变成另一个地址，而机器人固件仍访问旧地址，随后出现“检查新版本失败”或无法连接本地服务。

## 当前配置

- 机器人 MAC：`90:70:69:0e:a4:ac`
- 当前固件目标：`http://10.143.97.5:18003/a`
- 家长端：`http://127.0.0.1:8000/v2`
- 本地智能体运行中心：`http://127.0.0.1:8000/local-agent`

本地智能体页面会自动比较“电脑当前 IP”和“固件目标 IP”。显示“地址一致”时可以直接断开 USB 使用；显示“需要更新”时，应先处理地址变化。

## 推荐的稳定演示方式

优先在路由器中为电脑网卡设置 DHCP 地址保留，使电脑每次获得相同 IP。若比赛现场没有可控路由器，可使用固定手机热点，并保持电脑和机器人始终连接该热点。

临时换网后，可以重新构建并刷入新的 OTA 地址。后续产品化版本建议增加 mDNS 或 UDP 服务发现，让机器人自动寻找电脑，不再依赖写死的 IP。

## 开机顺序

1. 电脑和机器人接入同一网络。
2. 在电脑运行 `xiaozhi-server-sandbox/START_ALL.ps1`。
3. 打开本地智能体运行中心，确认五个服务在线且 IP 地址一致。
4. 给机器人独立供电开机。
5. 机器人开机询问学段后，即可语音选择小学、初中或高中模式。
