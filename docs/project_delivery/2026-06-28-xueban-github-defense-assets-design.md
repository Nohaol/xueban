## 视觉规范

- 继承现有商业计划书：暖白背景、深蓝标题，橙、绿、蓝为功能色。
- 使用细线、短色条、环形或双通道流程，避免连续横向同构流程图。
- 画布为 1920×1080，正文在投影环境下保持清晰。
- 每张图有一个明确结论，信息覆盖约 75%，避免大面积空白。
- 不使用虚构截图、虚构测试结果或无法验证的技术能力。

## 验收

- GitHub 分支可访问，Draft PR 包含完整说明和验证记录。
- 运行 `pytest` 与 JavaScript 语法检查。
- MCP endpoint 与 token 不进入 Git 历史。
- 每张 PNG 检查文字裁切、重叠、清晰度和色彩一致性。
- 最终 ZIP 作为本地备份，无论 GitHub 推送是否成功都保留。
