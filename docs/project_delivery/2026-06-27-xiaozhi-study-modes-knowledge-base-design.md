# 小智伴学三学段模式与知识库设计

## 1. 目标

在不刷写 ESP32 固件的前提下，先完成以下能力：

1. 一个小智智能体支持小学、初中、高中三种学习模式。
2. 每次设备开机后的第一次有效对话中，通过语音完成学段选择。
3. 小智明确播报当前模式，并在本次会话中保持该模式。
4. 家长端显示并可修改当前学段，提醒策略随学段变化。
5. 小智通过 MCP 将语音选择的学段同步到电脑端。
6. 建立一个同时覆盖小学、初中、高中的分层伴学知识库。
7. 摄像头自动提醒继续进入队列并保留主动播报扩展点，但本轮不解决待机状态下的无按键主动发声。

## 2. 范围与限制

- 本轮不创建三个独立智能体，继续使用智能体 `1817358`。
- 本轮不修改或重新烧录 ESP32 固件。
- “开机选择模式”定义为设备开机后，用户第一次按键或使用唤醒词进入对话时进行选择；设备不会在无人触发时主动开口。
- MCP 用于学段同步、提醒读取和状态记录，不宣称它可以从电脑端强制推送 TTS。
- 小智后台测试版只允许创建一个知识库并上传一个文档，因此所有学段内容必须放入同一文档。

## 3. 方案选择

### 3.1 推荐方案：单智能体动态学段状态

角色提示词维护一个 `study_stage` 会话状态，允许值为：

- `primary`：小学
- `middle`：初中
- `high`：高中
- `unset`：尚未选择

新会话中若状态为 `unset`，小智只询问一次学段。识别到“小学”“初中”“高中”或同义表达后：

1. 更新本次会话的 `study_stage`。
2. 调用 MCP 工具 `set_study_stage` 同步到电脑端。
3. 明确播报“已进入小学/初中/高中模式”。
4. 后续讲解、提问、鼓励与提醒均遵循该模式。

用户可以随时说“切换到小学模式”等指令重新选择。无法判断时不猜测，只给出三个选项再次询问。

### 3.2 未采用的方案

- 三个独立智能体：设备不能通过普通对话可靠切换绑定的智能体，维护和知识库关联也会重复。
- 自动推断年龄：容易因题目难度或语言习惯误判，且无法满足明确语音反馈的要求。

## 4. 三种学段行为

### 4.1 小学模式

- 语气：亲切、具体、鼓励式。
- 表达：每次一到两句话，一次只给一个步骤。
- 讲题：优先使用生活化例子，不直接给最终答案，先让孩子完成一个小步骤。
- 专注策略：容忍短时动作和视线偏移，连续分心约 20 秒后才进入提醒候选。
- 提醒策略：建议阈值 55 分，最短间隔 180 秒，10 分钟最多 2 次。
- 典型提醒：“我们先把眼睛放回题目上，完成这一小步就很棒。”

### 4.2 初中模式

- 语气：平等、清楚、适度鼓励。
- 表达：先确认目标，再拆分任务。
- 讲题：使用启发式追问，强调概念、条件和解题步骤。
- 专注策略：连续分心约 15 秒后进入提醒候选。
- 提醒策略：建议阈值 65 分，最短间隔 120 秒，10 分钟最多 3 次。
- 典型提醒：“先把注意力放回当前任务，我们完成这一小段再休息。”

### 4.3 高中模式

- 语气：简洁、尊重自主性，避免幼儿化表达。
- 表达：突出目标、时间和方法，不进行重复说教。
- 讲题：强调推理链、知识迁移、错因定位和复盘。
- 专注策略：识别标准更严格，但只有持续分心约 25 秒才提醒。
- 提醒策略：建议阈值 70 分，最短间隔 240 秒，10 分钟最多 2 次。
- 典型提醒：“回到当前目标，先完成这一题组，再统一复盘。”

## 5. 角色提示词结构

小智后台角色介绍包含以下规则，控制在平台字数限制内：

1. 身份与边界：桌面学习陪伴机器人，不替代教师或家长。
2. 新会话学段选择：状态未设置时先询问学段。
3. 语音确认：选择后必须播报当前模式。
4. 模式行为：分别定义表达难度、讲题方法和提醒方式。
5. 知识库调用：涉及学科知识、学习方法或分层讲解时，必须先调用 `search_knowledge`。
6. MCP 同步：选择或切换学段后调用 `set_study_stage`。
7. 提醒读取：仅在已经进入对话时调用提醒工具；没有提醒时不打扰孩子。
8. 安全边界：不羞辱、不贴标签、不把一次分心解释为学习态度问题。

## 6. MCP 工具设计

### 6.1 `set_study_stage`

输入：

```json
{
  "stage": "primary",
  "source": "voice"
}
```

行为：

- 校验学段值。
- 持久化到电脑端运行设置。
- 返回中文模式名称和对应提醒参数。
- 写入 MCP 调用日志。

### 6.2 `get_study_stage`

行为：

- 返回当前学段、中文名称、更新时间和提醒参数。
- 小智在模式不确定时可调用该工具恢复电脑端当前设置。

### 6.3 提醒工具调整

现有提醒项增加：

- `studyStage`
- `stageLabel`
- `policyVersion`

提醒话术由学段策略生成。提醒仍进入本地队列，并保留将来接入设备主动播报接口的能力。

## 7. 家长端设计

Devices 页面使用小学、初中、高中三段式控件显示当前模式。模式变化时：

1. 保存到后端运行设置。
2. 更新仪表盘顶部学生标签。
3. 更新托管提醒阈值、持续时间、冷却时间和频率限制。
4. Advice 页面生成与学段匹配的话术。
5. Review 页面在学习记录中标记学段，避免不同模式的数据混在一起解释。

电脑端手动切换和小智语音切换使用同一个后端接口，避免出现两个不同的当前模式。

## 8. 知识库设计

### 8.1 知识库名称

`小智分层伴学知识库`

### 8.2 知识库描述

```text
本文档包含小学、初中、高中三个学段的语文、数学、英语知识框架，
分层讲解方法、学习计划、专注提醒和复盘规则。当用户询问学科知识、
解题方法、学习计划或要求按学段讲解时，我不能直接作答，必须在每次
对话中调用该工具查询资料，并结合当前学习模式依据查询结果回答。
```

### 8.3 单文档结构

1. 使用规则与学段标签
2. 小学伴学规则
3. 小学语文、数学、英语核心方法与示例
4. 初中伴学规则
5. 初中语文、数学、英语核心方法与示例
6. 高中伴学规则
7. 高中语文、数学、英语核心方法与示例
8. 通用任务拆解、错题复盘和学习计划方法
9. 分心、疲劳、离座和坐姿提醒话术
10. 知识库调用验证题与唯一校验短语

回答学科问题时，查询词必须包含当前学段和学科，例如“初中 数学 一元二次方程 解题步骤”。

## 9. 数据流

### 9.1 语音选择模式

孩子进入对话 -> 小智询问学段 -> 孩子说“初中” -> 小智调用
`set_study_stage(middle)` -> 后端持久化 -> 家长端更新 -> 小智播报
“已进入初中模式”。

### 9.2 学科问答

孩子提问 -> 小智读取当前学段 -> 调用 `search_knowledge`，查询词带学段和学科
-> 按该学段的表达规则讲解 -> 通过追问确认理解，不直接包办作业。

### 9.3 专注提醒

摄像头分析 -> 学段提醒策略判断 -> 达到持续时间和阈值 -> 生成分层话术
-> 写入提醒队列 -> 已进入对话时由小智读取；待机主动播报留到后续设备方案。

## 10. 异常处理

- 未听清学段：再次列出三个选项，不修改当前模式。
- MCP 不在线：小智仍使用本次会话选定的模式，并说明电脑端同步暂未完成。
- 家长端与语音模式冲突：以后到达的有效设置为准，并记录来源和时间。
- 知识库未返回结果：明确说未检索到对应资料，再提供一般性思路，不虚构教材结论。
- 提醒过频：由学段冷却时间和 10 分钟频率上限拦截。
- 新会话未选择模式：不进入分层学科讲解，先完成模式选择。

## 11. 验收标准

1. 第一次有效对话会询问小学、初中或高中。
2. 三种回答都能得到准确的中文语音确认。
3. 语音切换后，家长端在一次状态刷新内显示相同模式。
4. 三种模式对同一道题给出明显不同但正确的讲解风格。
5. 学科问答记录中可看到知识库工具调用。
6. 使用验证题时能回答文档中的唯一校验短语。
7. 三种模式的提醒阈值、持续时间、冷却时间和话术符合本设计。
8. MCP 离线、知识库无结果和无效学段输入均有明确降级行为。
9. 本轮不以待机状态下的机器人主动发声作为验收条件。
